# 🏃🧠 운동 일지 개선 계획: 영구 기억 · 5일 이력 · 시계열 코칭

> 새 세션의 Claude에게: 이전 세션에서 사용자와 합의한 계획입니다.
> **먼저 아래 "현재 상태"를 코드로 확인**하세요. 상당 부분이 이미 구현돼 있어
> 처음부터 만들 게 아니라 **기존 위에 이어서/고치는** 작업입니다.
> 작업 전 반드시 읽을 것: `DEVLOG.md`, `app.py`, `workout_flow.py`,
> `workout_agent.py`, `whoop_agent.py`, `core/profile.py`, `notion_agent.py`.

## 배경 (사용자 요구)

1. **운동 프로필이 저장이 안 된다** — Streamlit Cloud는 파일이 영구 저장되지
   않아서(재배포·잠들면 사라짐) 프로필/기억이 날아간다.
2. **내 성향이 다음 일지에 반영돼야 한다** — 수정 요청·코치 답장에서 드러난
   내 문체·습관(예: 명상 매일, 무릎 주의)을 다음 날에도 기억해 반영.
3. **최근 5일치 운동이 보여야 한다** — 일지를 못 쓰는 날이 있고, 나중에 지난
   날 운동으로 일지를 쓰려면 그 기록을 골라야 한다.
4. **코치와의 대화가 시계열로 쌓여야 한다** — 그래야 코칭이 "이어지는 대화"로
   제대로 발전한다.

## 현재 상태 (이미 구현된 것 — 확인 후 재사용/확장)

- **프로필 Notion 백업**: `core/profile.py`의 `save()`가 Notion에 백업.
  `notion_agent.has_settings_credentials()` / `save_settings()` /
  (로드 경로) 존재. `app.py`의 `_save_profile()`가 백업 성공 여부를 안내.
  → **"저장 안 됨"의 실제 원인은 Notion 설정 백업 미구성일 가능성**이 큼.
  먼저 이게 실제로 동작하는지(NOTION_TOKEN 및 설정 저장 위치) 진단할 것.
- **문체 기억(style_memory)**: 프로필에 `style_memory` 필드
  (`app.py` "🧠 기억된 문체 취향"), `workout_agent.update_style_memory(
  existing, feedbacks)`로 피드백을 누적 요약해 저장. `workout_agent`가
  글 생성 시 이 값을 반영(313행 부근).
  → **2번 요구의 뼈대는 이미 있음.** 문체뿐 아니라 습관·주의사항까지
  포괄하도록 확장하고, "코치 답장(user_note)"도 여기에 축적되게 연결.
- **최근 운동 조회**: `whoop_agent.get_recent_workouts(days=2, limit=25)`.
  → 5번 요구는 **`days=2` → `days=5`** 로 늘리고, w0 화면에서 **날짜별로
  그룹핑**해 "오늘/어제/N일 전"을 골라 쓸 수 있게 UI 보강.
- **추세 코칭**: `whoop_agent.get_trend_summary()` 로 최근 숫자 추세 제공(2주).

## 해야 할 일 (제안)

### A. 프로필/기억 영구 저장 검증·수리 (요구 1, 2)
- Notion 백업이 실제로 저장·복원되는지 끝까지 확인. 안 되면 원인 수리.
- `style_memory`를 "문체 + 습관 + 주의사항"으로 확장. 코치 답장(`wk_coach_notes`)
  에서 드러난 지속적 사실(명상 습관 등)도 프로필에 축적.
- 저장 위치는 **Notion**(이미 연결됨)을 1순위로. (repo에 JSON 커밋은 지양)

### B. 최근 5일 운동 선택 (요구 3)
- `get_recent_workouts(days=5)`. w0에서 날짜 헤더로 그룹핑, 과거 날짜 운동도
  선택 가능하게. 일지 제목/집계의 "오늘" 가정이 과거 날짜에도 맞게 조정.

### C. 시계열 코치 로그 (요구 4) — 이번 작업의 핵심(신규)
- 운동 일지를 발행할 때마다 **날짜별 코치 로그**(그날 요약·코치 분석·내
  답장)를 Notion에 append. (별도 DB 또는 페이지, `NOTION_*_PARENT_ID`)
- 다음 일지 분석 시 최근 며칠치 코치 로그를 불러와 `analyze_workout`에
  넘겨서 "지난번에 이렇게 얘기했었죠" 식으로 이어지게. (토큰 절약 위해
  전체가 아니라 압축 요약만 — get_trend_summary 패턴 참고)

## 주의 (이전 세션의 교훈)

- Streamlit 위젯 값은 렌더 후 코드로 바꾸면 크래시 → 버튼을 위젯 앞에
  두고 값 주입은 렌더 전에 (`devlog_flow.py` 참고).
- 세션 상태는 네트워크 끊김에 사라짐 → `core/draft.py` 초안 저장 유지.
- 사이드바 `APP_VERSION`을 올려야 배포 반영을 눈으로 확인 가능.
- 긴 글 revise는 `max_tokens` 자동 확대 (`core/writer.py`).
- **main에는 invest 세션도 푸시 중** — 충돌 방지 위해 이 작업은 **별도
  브랜치 → PR**로 진행 권장. `app.py`를 invest 세션과 동시에 편집하지 말 것.
