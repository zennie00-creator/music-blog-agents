# 📈 다음 프로젝트: 투자 일지 모드 (새 세션용 계획서)

> 새 세션의 Claude에게: 이 파일은 이전 세션에서 사용자와 합의한 계획입니다.
> 기존 앱 구조(아래 참고)를 그대로 활용해 네 번째 모드를 추가하는 작업입니다.

## 배경

- 사용자는 기존에 make.com으로 투자 정보 워크플로우를 운영:
  구글시트(마켓인덱스·주식·채권 수집) → Perplexity(1차 분석) → Gemini(테크
  시장과 함께 심화 분석) → Gemini(종합 정리) → Slack 전송.
- 문제: **깊이가 없음.** 매번 백지에서 분석하고, 사용자 본인의 포트폴리오
  맥락과 과거 판단이 반영되지 않기 때문.
- 결론: 이 저장소의 "일지 에이전트"(음악/운동/개발 모드)에 **투자 일지
  모드**로 통합한다.

## 설계 방향 (합의됨)

1. **시장 데이터 수집** — make.com/구글시트 대신 파이썬에서 직접.
   yfinance 등 무료 API로 지수·주식·채권·환율. (LLM 비용 0)
2. **투자 분석 엔진 = Grok (xAI API)** — 사용자가 명시적으로 원함.
   X(트위터) 실시간 정보 접근이 테크 동향에 유리.
   OpenAI 호환 REST API, 키는 Streamlit Secrets에 `XAI_API_KEY`.
3. **일지 작성 = Claude** — Grok의 시장·테크 분석 + **사용자 본인의 판단
   메모**(운동 일지의 '운동 전/중/후 기분'에 해당)를 합쳐 일지로.
4. **발행** — 기존 Notion 자동 발행(notion_agent) + 네이버 복사 재사용.
   투자 일지용 별도 Notion 부모: `NOTION_INVEST_PARENT_ID`.
5. **깊이의 원천** — 포트폴리오/관심종목 프로필(core/profile.py 패턴),
   과거 일지 누적(Notion), 운동 모드의 '코치에게 답장' 같은 대화형 보정.
6. **자동 매매 아님** — 분석·기록 보조 도구로만 설계.

## 기존 구조 (재사용할 것)

- `app.py` — 모드 선택 대문 + 세션 상태. 모드 추가 방법은 devlog 모드
  추가 커밋 참고.
- `core/writer.py` — Claude 호출 공통 (generate / revise_with_feedback)
- `core/profile.py` — 프로필 저장 패턴 (투자 프로필 추가)
- `core/draft.py` — 네트워크 끊김 대비 초안 자동저장 ("이어서 하기")
- `core/naver_html.py` — 네이버용 HTML/텍스트
- `notion_agent.py` — Notion 발행 (페이지/DB 감지, md 블록 변환, 이미지
  업로드, parent_id 지정 가능 — 투자용 부모 ID만 넘기면 됨)
- `workout_flow.py` — 흐름 설계의 표준 예시 (데이터 수집 → 사용자 입력 →
  분석 → 수정 요청/답장 → 발행)
- `DEVLOG.md` — 지금까지의 개발 이력

## 주의 (이전 세션의 교훈)

- Streamlit 위젯 값은 위젯이 그려진 뒤 코드로 바꾸면 크래시 —
  버튼을 위젯 앞에 배치하고 값 주입은 렌더 전에 (devlog_flow.py 참고)
- 긴 글 revise는 max_tokens 자동 확대 필요 (core/writer.py에 이미 반영)
- 세션 상태는 네트워크 끊김에 사라짐 — 새 모드도 core/draft.py로 초안
  저장 통합할 것
- 사이드바 APP_VERSION을 올려야 사용자가 배포 반영을 확인 가능
